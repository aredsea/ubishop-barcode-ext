> **ISOLATED content script 에서 `javascript:` 링크를 `.click()` 하면 실행되지 않는다.**

`chrome.scripting.executeScript({world:'MAIN'})` 으로 해야 한다. 계정전환 자동화가 이미 같은
이유로 이 경로를 쓴다(`background.js:296~`).

**더 아픈 건, 우리 코드가 이미 주석으로 반박하던 걸 설계했다는 점이다:**

| 폼 전체 직렬화 후 직접 POST | `skin.js:717` *"DOMParser 후 form.elements 로는 hidden 이 form1 밖으로 빠져나온다(실측)"* |

## 실측으로 확보한 ERP 쓰기 계약 (스펙 1절에도 있음)

전송을 차단한 상태로 캡처했고, 사후에 대상 레코드를 재조회해 **실제로 안 바뀐 것**까지 확인했다.

```
standby   → POST /jun/orderitem/orderItemStandby.do
             쿼리: status1=<목표> status2=<요구 현재상태> sKey=<페이지별 토큰> + 검색조건 24종
             본문: idx=<orderSeq>        ← 체크된 것 정확히 하나 (C 의 전제조건 충족)

setCurrent → 폼이 아니다. location 대입(GET 이동)
             /jun/orderitem/orderItemPopCurrentSettingModify.do?...&barcode=<선택>
```

- **`sKey` 는 페이지 로드마다 재발급된다.** 재사용 불가 — 갓 로드한 문서에서 얻어야 한다
- **⚠⚠ `setCurrent` 는 GET 이 쓰기다.** `…Modify.do` 를 "확인용으로 한 번 여는" 것 자체가 배정이다.
  읽기는 `…ModifyForm.do`(Form 접미사). **한 글자 차이**라 `tests/auto-orchestrator.test.js` 로 못박아뒀다
- 수정폼 필드: `textarea[shopRemark]`=주문비고, `textarea[remark]`=발주비고 → `POST orderItemModify.do`
- 상단 [본사확인]/[본사확인취소] 는 **ERP 자체 버튼**이고 라벨이 `<img>` 라 텍스트 검색에 안 잡힌다.
  둘 다 `TD.left` 안 → 신규 버튼 삽입 지점
- **사이트 인코딩이 섞여 있다** — 수정폼 UTF-8, 배정 팝업 EUC-KR

## 새로 당한 함정 (이전 문서 목록에 추가)

- **`setCurrent` 계측 중 실제 쓰기가 나갈 뻔했다.** `form.submit` 만 stub 했는데 그 함수는
  `location` 을 쓴다. iframe 을 즉시 제거한 덕에 우연히 안 나갔다(내가 넘긴 바코드가 후보에
  그대로 남아 있어 미반영 확인). → **write-free 프로브는 `location` 대입·`assign`/`replace`·
  `window.open`·`form.submit`·`requestSubmit`·앵커 클릭을 전부 막고, 사후 재조회로 무변경을 확인한다.**
- **PowerShell `Set-Content -Encoding utf8` 이 테스트 파일 한글을 통째로 깨뜨렸다**(BOM + cp949 읽기).
  이 저장소가 이미 문서로 경고한 함정인데 또 걸렸다. **소스 편집은 Write/Edit 도구나 node 로.**
- **`git show :<path>` 로 미스테이징 파일을 "복원"하면 작업분이 날아간다.** 인덱스 = HEAD 라
  126행을 잃었다(백업으로 복구). 임시 변이 실험은 파일 백업을 먼저 뜨고, 복원도 그 백업으로.
- **배정 후보 목록은 살아있다.** 같은 주문의 후보가 30분 사이 144→142건, 첫 바코드가
  `2604O7`→`2604O9` 로 바뀌었다(그 사이 매장 직원이 정상 처리). "첫 행"은 실행 시점 값이지
  고정값이 아니다.
- **`manifest.json:90` 의 `skin.js` 는 `all_frames:true` 인데 프레임 가드가 없었다.** 주문전표
  페이지엔 ERP 자체 iframe 이 10개 있다. v3.7.2 에서 파일 최상단에 가드를 넣었다 —
  `ensurePageSizeAtStart`(document_start 에 `location.replace`)보다 **앞**이어야 한다.
