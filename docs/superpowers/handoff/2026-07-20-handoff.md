## 이번 세션에 한 일

### 1. 상품집계 판매탭 '이익율' (app v2.8.0 → 2.9.0) — **완료·라이브 검증됨**
요구: 판매탭(`sheetStatisList.do?tcode=statis_sheet_barcode`)의 `연결번호` 열을 `이익율`로 교체.
`이익율 = (실판매가 − 총입고가) ÷ 실판매가`. **0% 는 미표기**(사은품이 많아 가독성). 통계에도 반영.

- 본표: 연결번호 → 이익율. **라이브 41건 전부 검산 일치, 빈칸 59건은 전부 실판매가 0.**
- 통계 오버레이 `renderOrder` 3뷰(직원별·제품별·매장별): 그룹행·상세행·요약·XLSX 에 이익율.
- 헤더 바에 **정렬 컨트롤**(기준 선택 + 오름/내림). 표 헤더를 뷰마다 손대는 대신 컨트롤 하나로
  세 뷰에 일관 적용 — 표 구조를 안 건드려 회귀 위험이 낮다.
- 집계는 '비율의 평균'이 아니라 **합계끼리**(Σ실판매가, Σ총입고가). 요약 이익율 50.6% 가
  **ERP 자체 합계행**(총입고가 224,912,477 / 실판매가 454,981,298)으로 계산한 값과 일치 — 독립 교차검증.

### 2. ⚠ 앞선 커밋(v2.8.0/2.8.1)의 '통계 반영' 주장은 **사실이 아니었다**
`statis.js` 의 `render(result, meta)` 는 **호출부가 0개인 죽은 코드**다. 판매탭이 실제로 그리는 것은
v2.7.0 에서 갈라져 나온 **`renderOrder(items, meta)`**(호출부 2곳). 죽은 쪽을 고쳐놓고 "통계 반영"이라
커밋했고, **라이브 오버레이를 열어보고서야 발견**했다. 죽은 쪽 편집은 원복했고 죽은 함수는 그대로 남아 있다.
→ **`statis.js` 를 손댈 때 `render` 와 `renderOrder` 를 반드시 구분할 것.**

### 3. 고친 자기 버그 — 상세행 colspan
이익율 열을 늘리면서 상세행 `colspan` 을 안 올려 판매탭 3뷰의 펼침 행이 한 칸씩 모자랐다(표가 밀려 보임).
7/9/7 로 보정. 소스에서 헤더 `<th>` 수를 직접 세어 `1 + colspan` 과 대조하는 **회귀 테스트**를 넣었고
red-green(버그 되돌리면 실패)까지 확인. `tests/statis-profit.test.js` 13 pass.

### 4. 배포 인덱스 툴링의 조용한 손상 (같이 고침)
`build-app-index.ps1` 이 Windows PowerShell 5.1 에서 `app-files.json` 을 **조용히 망가뜨렸다**:
- `Get-Content -Raw` 가 BOM 없는 UTF-8 을 cp949 로 읽어 **note 의 한글 전부 모지바케**
- `Set-Content -Encoding UTF8` 이 **BOM** 을 붙여 loader 의 `JSON.parse` 를 깨뜨림
- `SHA256::HashData` 는 .NET 5+ 전용이라 5.1 엔 **없음**(해시 전부 빈 값인데 "N files" 성공 메시지)

셋 다 실패해도 계속 진행해 성공처럼 보이는 게 제일 위험했다. → 구현을 **`build-app-index.js`(node)** 로
옮기고 `.ps1` 은 shim. 이제 **파일 누락·단독 CR 이면 쓰지 않고 exit 1**. `build-shell-index.ps1` 의
BOM·HashData 도 같이 수정. (shell 인덱스는 이번에 실행하지 않았음 — SHELL 변경이 없어서.)

## 성능 관련 이미 확인된 사실 (다시 조사하지 말 것)
- `rules/cache.json`(DNR `modifyHeaders`)이 서버의 `max-age=5` 를 이미지 24h / 스크립트 1h 로
  **이미 덮어쓰고 있고 작동 중**. 콜드 1674ms → 재로드 708ms, 이미지 1012ms → 1ms.
- 따뜻한 캐시에선 **TTFB 544ms 가 전체의 77%** — 확장이 더 짜낼 여지가 사실상 없다.
- `match_about_blank` 이 없어 about:blank 프레임엔 주입되지 않는다("11× 파싱"은 이전 세션의 오해).

## 이 프로젝트에서 반복해서 당한 함정
- **PowerShell 5.1** — `SHA256::HashData` 없음 / `Set-Content -Encoding UTF8` BOM /
  `Get-Content -Raw` ANSI 읽기. 전부 **조용히** 실패한다. 인덱스는 node 로.
- **Bash 도구 heredoc 이 백슬래시를 한 번 삼킨다** — python 안에서 `\\r?\\n` 이 실제 CR/LF 가 되어
  정규식이 깨졌다. 정규식·이스케이프가 든 파일 편집은 **Edit/Write 도구나 PowerShell** 로.
- **유비샵 금액셀은 값이 구분자 없이 붙어 나온다**(`878,439878,4390`). 첫 값만 집어야 한다(`firstNum`).
- **`render()` 가 innerHTML 을 갈아끼우므로** 검증 스크립트에서 노드를 미리 잡아두면 분리된 표를
  계속 읽는다. 이번에 이걸로 "정렬 결함"을 오보고했다가 정정했다. **매번 새로 조회할 것.**
- **크롬 팝업 정리 시 창 크기 필터를 넓게 잡으면 목록 창까지 닫힌다**(실제로 크롬을 통째로 닫았음).
